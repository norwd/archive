SCC toolchain

Verifying integrity

The SCC releases are signed using the project signify key. You can validate the sign using the OpenBSD signify program. If your system does not provide a compatible signify program, you can use this.

Releases

  • scc-0.1.tar.gz (sig):

    This is the initial release of SCC. Many things have to be done to have an usable toolchain, but this release already contains many important bits and it is able to compile complex enough programs like for example dwm.

    The code generated is known to have some errors, the elf support is not finished, as only generates raw bytes without any binary format (elf, coff, ...) and some of the tools like ld or objcopy are just a draft or directly a placeholder.
