This program is something I thought to be impossible. My sincere congratulations to the author, Hisashi Iizawa, who has proven me wrong and made me swallow my own words. (My main objection was that the initialization subprogram could require more than the maximum memory size; this is obviously wrong as this program shows but is not that far from reality as discussed below.)

Those who are still skeptical with respect to whether this file has the whole lyrics harcoded or actually does loop as it claims, consider the following facts:

1) The program length is 22561 instructions; its output is 11459 bytes long. The ratio is less than 2 instructions per output byte, clearly insufficient <i>with Malbolge's limitations</i> for a program which simply outputs constant text. Compare this to the other "fake" version, known to be constant text output: 7775 instructions for a 1175-byte output, about 6.6 instructions per output byte, much more expectable for a constant-text output program. I think that a 999-bottles-of-beer version would not be too difficult to write for the author to demonstrate this point without a shadow of doubt (or a version which uses a different base; e.g. ZZ bottles of beer through AA bottles of beer would output about 77000 bytes and be very similar in length, while proving without a doubt that it performs conditional jumps.)

2) I've uploaded a normalized version of this program here:

http://www.formauri.es/personal/pgimeno/temp/bottles-995.n

An examination of this program reveals that the initialization phase is the code between position 99 and position 22043 inclusive so it is 21945 bytes long (there's no jump or output instruction there; the only instructions are j, o, p and *, i.e. data manipulation). Quite expectable for a complex Malbolge program.

3) It takes 13,802,606 instructions executed to print the lyrics and stop. That's not the typical constant-text program either, which basically executes the instructions needed without looping. I wouldn't be surprised if the core is an interpreter for a simple language and 99bob is written in that language.

In my opinion, this version deserves a much higher score than it currently has. A piece of fine art. Congratulations.
