Compiling 'C' programs on Eidolon

This document briefly describes how we've configured the GNU CC compiler, on Eidolon. By default GNUC CC compiles your C code into linux ELF objects, which then get linked into ELF executables. GNU CC can also be used to compile code meant to be run on some other type of machine. In fact, several different configurations of GNU CC, for different target machines, can be installed side by side. On Eidolon, we've configured GNU CC to compile for the following targets:

  • i486-linux [default]
  • i486-linuxaout
  • sparc-sun-sunos4.1.4
  • mips-dec-ultrix4.2
When you run /usr/bin/gcc, you're not actually running the compiler, you're running what they call the 'driver program'. The driver is responsible for figuring out which preprocessor, compiler, assembler, and linker to run to build your executable. The -b switch to GNU CC controls which target it compiles for.

Examples: 
	% gcc test.c		/* Compile for the default target	*/
	% gcc -b aout test.c	/* Compile for i486-linuxaout		*/
	% gcc -b mips test.c    /* Compile for mips-dec-ultrix4.2	*/
	% gcc -b sparc test.c	/* Compile for sparc-sun-sunos4.1.4	*/
