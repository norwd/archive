Quick and Dirty Guide to Page Publishing

Step 1: Your "Web directory"
In your home directory, there should be a directory called ".html" - if it doesn't exist, then create it (using the mkdir command). Make sure that it is world-readable (using the chmod command - "chmod a+r .html").

Step 2: The Default Web Page
There should be a document in your .html directory called "home.html" - this is the default document that the Web server will look for if no specific document is called. Make sure that your top-level, or "home" page is called "home.html".

As always, make sure that it is world-readable. Use the chmod command as shown above.

Step 3: Revenge of the CGI Script
Since Eidolon is so cool, users are free to write and run their own CGI scripts. CGI scripts are those cool, simple little things that handle HTML forms and the like. To let the Web server know that you have a CGI script, name your script scriptname.cgi. That's all there is to it.

Tutorials on CGI scripting can be found at foo.

Step 4: How to Access Your Page
If you wanted to post your URL to Yahoo! or Lycos, you would write it like so:

    http://www.muppetlabs.com/~your_user_name

Step 5: Other Considerations
Let the world know just how cool you are - be sure to create subdirectories in your .html directory, especially for things like images and sound files. It'll keep your directories clean and orderly, and will demonstrate your level of Clue to the rest of the 'net community.
